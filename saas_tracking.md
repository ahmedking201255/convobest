### 5️⃣ المرحلة 5: أدوات العميل - حملات الإرسال الجماعي وبوت الذكاء الاصطناعي (مكتملة ✅)

    2. إعداد مسار ضبط الشات بوت لتوفير الردود التلقائية بالذكاء الاصطناعي (OpenAI و Gemini).
    3. تعديل مستمع الويب هوك الخاص بالرسائل الواردة لتوليد وإرسال الردود الذكية وتخزينها تلقائياً.
    4. تفعيل جدولة الحملات الإعلانية ومحرك الإرسال في الخلفية مع مانع الحظر التلقائي (Anti-Ban).

### 6️⃣ المرحلة 6: نظام الدفع والاشتراكات التلقائية المخصصة (مكتملة ✅)
*   **الإجراءات المتخذة**:
    1. تم إضافة جدول المعاملات المالية `Transaction` لتخزين المعاملات المبدئية وحالات التحققات.
    2. بناء مسار API الفوترة وتسجيل طلبات الدفع المعلقة `/api/billing/checkout` لتأكيد ومتابعة الحالة.
    3. بناء خادم استقبال إشارات وتأكيد رسائل التحويل الهاتفي `/api/billing/sms-gateway` مع نظام فحص ومطابقة ذكي بالرمز السري للمحفظة أو اسم حساب إنستاباي.
    4. تصميم واجهة الفوترة والترقية البصرية للمشتركين `/dashboard/billing` التي تدعم الدفع والانتظار التفاعلي مع تفعيل الحسابات فورياً.
    5. كتابة الكود المصدري البرمجي لتطبيق الأندرويد المساعد بلغة **Kotlin** لقراءة وتحليل رسائل التحويل (SMS Reader).

### 7️⃣ المرحلة 7: لوحة تحكم المسؤول العام (Admin Dashboard) (مكتملة ✅)
*   **الإجراءات المتخذة**:

    2. توفير أدوات لتأكيد المعاملات المالية يدوياً في حالة فشل المطابقة التلقائية.
    3. تفعيل إمكانيات تعديل باقات المستخدمين وتفاصيل الجلسات مباشرة من لوحة المدير.

### 8️⃣ المرحلة 8: الاختبارات الشاملة وتحسين الهوية البصرية (مكتملة ✅)
*   **الإجراءات المتخذة**:
    1. إصلاح تداخل نصوص حقول الإدخال والرموز التوضيحية لصفحات التسجيل والدخول.
    2. إعادة توزيع وتنسيق خيار "نسيت كلمة المرور" وبناء نظام الاستعادة الكامل بالرمز السري المؤقت OTP.
    3. التحقق من سلامة البناء البرمجي باستخدام `npx tsc --noEmit` لضمان خلو الأكواد من أي أخطاء.

### 9️⃣ المرحلة 9: صندوق الوارد الموحد (الدردشة التفاعلية كـ WhatsApp Web) (مكتملة ✅)
*   **الإجراءات المتخذة**:
    1. **بناء واجهة الدردشة (UI/UX)**: تصميم واجهة تحاكي واتساب ويب بالكامل (قائمة الدردشات النشطة مع البحث، شاشة المحادثات المتتالية، وشريط إرسال الرسائل الفورية) في المسار `/dashboard/inbox`.
    2. **تطوير مسارات الـ API**:
       - إنشاء مسار جلب قائمة الدردشات الفريدة النشطة `/api/chat` مجمعة حسب رقم الهاتف وآخر رسالة.
       - إنشاء مسار جلب سجل الرسائل المتبادلة `/api/chat/messages` لجهة اتصال محددة.
       - إنشاء مسار إرسال الرسائل الفورية `/api/chat/send` وربطه بمحرك Go للواتساب وتخزين الرسائل الصادرة بجدول `MessageLog`.
    3. **التحديث والتزامن اللحظي (Polling)**: ربط الشاشة بنظام تزامن دوري يضمن جلب التحديثات والرسائل الجديدة كل 3 ثوانٍ دون الحاجة لتحديث الصفحة.
    4. **مكتبة الصح المزدوج والوسوم**: إدراج علامات الصح المزدوجة وإحصائيات المحادثة في لوحة العميل.
    5. **مزامنة جهات الاتصال (Contacts Sync)**: تطوير مسار API خاص `/api/chat/sync` وزر مزامنة في واجهة المستخدم لاستيراد كافة جهات الاتصال النشطة من الهاتف وعرضها فوراً في الصندوق الوارد لتسهيل بدء المحادثات.

### 🔟 المرحلة 10: التكامل المباشر مع متاجر ووكومرس (WooCommerce) (مكتملة ✅)
*   **الإجراءات المتخذة**:
    1. **تحديث قاعدة البيانات**: إضافة وتزامن جدول `WooCommerceConfig` لتخزين إعدادات الربط وحالة الخدمة وقوالب الرسائل (قيد الانتظار، جاري التحضير، مكتمل) لكل رقم واتساب.
    2. **بناء خوادم الـ API**:
       - إنشاء مسار جلب وتعديل إعدادات وقوالب ووكومرس المخصصة لكل جلسة واتساب `/api/integration/woocommerce/config`.
       - إنشاء مسار استقبال ويب هوك الأحداث من ووردبريس `/api/integration/woocommerce` ومطابقتها بالقوالب واستبدال المتغيرات ديناميكياً ثم إرسالها للواتساب فورياً عبر محرك Go وحفظ السجلات.
    3. **بناء إضافة ووردبريس وحزمها**: كتابة كود إضافة الووردبريس بلغة PHP وضغطها تلقائياً بملف ZIP قابل للتنزيل المباشر `portal/public/downloads/evopulse-woocommerce.zip`.
    4. **شاشة الربط في لوحة التحكم**: تصميم واجهة إعدادات ووكومرس بمسار `/dashboard/woocommerce` لإدارة الحالة، وتعديل القوالب، ونسخ المفاتيح، وتنزيل الإضافة مع دليل تثبيت كامل.

### 1️⃣1️⃣ المرحلة 11: التكامل والربط المباشر مع جداول بيانات جوجل (Google Sheets) (قيد التخطيط 📋)
*   **خطة التنفيذ ومراحل التشغيل**:
    1. **تكامل واجهة برمجة تطبيقات Google Sheets API**:
       - ربط خادم المنصة بمشروع Google Cloud Console وتفعيل Google Sheets API و Google Drive API.
       - تفعيل مصادقة OAuth 2.0 الآمنة ليتمكن العميل من ربط حسابه في جوجل والسماح بقراءة ملفات الجداول الخاصة به بأمان.
    2. **بناء مستورد البيانات الذكي (Data Mapper)**:
       - تطوير واجهة مستخدم تتيح استعراض ملفات جداول جوجل واختيار الورقة (Sheet) المستهدفة.
       - قراءة الصف الأول كأعمدة والسماح للمستخدم بمطابقة الأعمدة (تحديد عمود رقم الهاتف، عمود الاسم، وأعمدة المتغيرات الإضافية).
    3. **محرك الحملات وجدولة الإرسال المتتابع**:
       - صياغة الرسائل باستخدام متغيرات الأعمدة ديناميكياً (مثال: "مرحباً `{الاسم}`، نود تذكيرك بموعدك يوم `{اليوم}`...").
       - إرسال الرسائل عبر خادم الجدولة بالخلفية لضمان تفعيل محاكي السلوك البشري وفواصل التأخير (Anti-Ban).
    4. **إضافة Google Sheets المخصصة (Workspace Add-on)**:
       - بناء إضافة ووردبريس/جوجل (Google Workspace Add-on) يتم تثبيتها مباشرة في شريط أدوات جداول جوجل.
       - تمكن الإضافة المستخدم من إدخال مفتاح الـ API وتحديد الأرقام وإطلاق الإرسال الجماعي مباشرة من داخل الجدول دون مغادرته.
